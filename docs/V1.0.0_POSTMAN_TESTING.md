# v1.0.0 Postman testing

Set `baseUrl` to `http://localhost:5000/api` and send `Authorization: Bearer <access-token>` (or use the login cookie). Replace angle-bracket values with real MongoDB ObjectIds.

## Employee profile

| Request | Method and URL | Body / expected result |
| --- | --- | --- |
| Get profile | `GET {{baseUrl}}/users/me` | `200` with populated `departmentId` and `managerId`. |
| Update profile | `PUT {{baseUrl}}/users/me` | JSON: `{ "phoneNumber": "+919876543210", "address": "Bengaluru", "gender": "prefer_not_to_say", "bloodGroup": "O+" }`; expect `200`. Protected fields such as `role` return `400`. |
| Upload profile picture | `PUT {{baseUrl}}/users/me/profile-picture` | form-data field `profilePicture` (JPG/PNG up to 5 MB); expect `200`. A replacement removes the prior Cloudinary asset. |

## HR/Admin employee management

| Request | Method and URL | Body / expected result |
| --- | --- | --- |
| List/filter | `GET {{baseUrl}}/users?page=1&limit=20&search=ana&role=employee&departmentId=<departmentId>&isActive=true&managerId=<managerId>` | HR/Admin only; expect `200`. Every filter is optional. |
| Employee details | `GET {{baseUrl}}/users/<employeeId>` | HR/Admin: `200`. A manager may retrieve only a direct report; an employee only themself (`403` otherwise). |
| Update profile | `PUT {{baseUrl}}/users/<employeeId>` | HR/Admin JSON: `{ "joiningDate": "2026-01-15", "phoneNumber": "+919876543210" }`; expect `200`. |
| Deactivate/activate | `PUT {{baseUrl}}/users/<employeeId>/status` | `{ "isActive": false }` then `{ "isActive": true }`; expect `200`. Self-deactivation is `400`; final active admin is `409`. |
| Assign manager | `PUT {{baseUrl}}/users/<employeeId>/manager` | `{ "managerId": "<managerId>" }`; expect `200`. Send `null` to unassign. Self, inactive, nonexistent, or circular manager assignments return `400`/`409`. |
| Change department | `PUT {{baseUrl}}/users/<employeeId>/department` | `{ "departmentId": "<departmentId>" }`; expect `200`. An inactive/nonexistent department returns `400`. |
| Change role | `PUT {{baseUrl}}/users/<employeeId>/role` | Admin only; `{ "role": "manager" }`; expect `200`. Removing the final active admin returns `409`. |

## Department management

| Request | Method and URL | Body / expected result |
| --- | --- | --- |
| Create | `POST {{baseUrl}}/departments` | HR/Admin JSON: `{ "name": "Engineering", "description": "Product engineering", "managerId": "<activeUserId>" }`; expect `201`. Duplicate name: `409`; invalid/inactive head: `400`. |
| List/filter | `GET {{baseUrl}}/departments?page=1&limit=20&search=eng&isActive=true` | Authenticated user; expect `200`. All filters optional. |
| Get | `GET {{baseUrl}}/departments/<departmentId>` | Authenticated user; expect `200`. |
| Update | `PUT {{baseUrl}}/departments/<departmentId>` | HR/Admin JSON: `{ "description": "Updated", "isActive": true }`; expect `200`. Deactivating while active employees remain returns `409`; reassign/deactivate them first. |
| Delete | `DELETE {{baseUrl}}/departments/<departmentId>` | Admin only; expect `200`. A department with any employee reference returns `409`. |

All malformed IDs and invalid body values return `400`; missing resources return `404`; authentication failures return `401`; role/scope failures return `403`.
